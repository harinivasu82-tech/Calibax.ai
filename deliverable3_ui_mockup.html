Four intelligence chips surface automatically before the CEO types anything — NZ margin alert, Auckland DC delay, SG e-commerce, AU hypermarkets.

02Structured data response — US-01
Prototype scope Screen 2 of 3 · Structured Query Response

Layer 1: Financial data Q&A with source attribution. The CEO asks a margin question. The system returns blended prose and three data cards (March margin, Q3 average, variance) with a collapsed source attribution panel. The input shows the follow-up being typed — contextual chips replace idle chips.

① Data cards — structured answer
Three metric cards: 18.4% ↓ (March), 21.6% (Q3 baseline), −$1.2M (variance). Values from the pre-computed lookup fixture, verified by the numeric guardrail.
US-01
② Source panel — collapsed default
"Sources · 3 sources" is collapsed by default. CEO expands on click. Never zero sources on a successful response — the trust signal that addresses the Tableau failure.

03Cross-signal synthesis — the wow moment
Prototype scope Screen 3 of 3 · Blended Cross-Source Response

SAP + Teams + Email joined in one response. The CEO asks whether the NZ margin issues are connected to regional team signals. The system returns one blended response joining SAP financial data with a Teams escalation, cites Marcus Reid by name, and shows 4 expanded source pills. No competitor produces this response format.

① Cross-signal synthesis block
Green-tinted card labelled "Cross-signal synthesis · SAP + Teams + Email". Two source pills — [SAP · Finance] ⟷ [Teams · Regional Ops] — connected by arrow. Causal narrative inside the block.
US-03
② Source panel — expanded, 4 sources
Panel open state. Four sources type-coded: SAP (blue), Teams (purple), Email (amber), WMS (green). Shows system type and source name. Maximum trust signal at the wow moment.
US-04
③ Named actor — grounded and traceable
Marcus Reid (NZ Regional Ops Lead) cited by name from the Teams corpus. CEO can ask a follow-up targeting that person directly. Grounded, verifiable, actionable.

04Design system — tokens for the engineer

All three screens share a unified dark token system. The engineer must not deviate from these values when wiring the DS backend to the frontend. Source type colour-coding must be applied consistently across all response components.

Color tokens
#0d0f0d — background
#141714 — sidebar / cards
#1c201c — elevated / hover
#23c45e — primary accent
#e05050 — error / declining
#e8a020 — tertiary / warnings
#5b9cf6 — secondary / info
Typography
Fraunces italic
Display headings, AI response name
DM Sans 500
Body text, navigation labels
DM Mono
IDs, timestamps, source labels, metric values
Source type colour coding
SAP · blue Teams · purple Email · amber WMS · green
Type-coding must be consistent across all response cards, source panels, and synthesis blocks. Never swap colours between types.
